JavaScript Kodu Nasıl Çalışır

JavaScript yorumlayıcıları web tarayıcı (Google Chrome, Mozilla vb) entegre edilmiştir. Bu sayede tarayıcılar kodu anlar ve yürütülme işlemini yapar. JavaScript kodlarının özelliğinden bir tanesi tek iş parçacıklı (single-threaded), senkrondur, ama asenkron davranışlara da sahiptir. Kodları adım adım çalıştırmasıdır.

function greetings(name) {
    console.log("Merhaba", name)
}
                                
let name = "Brendan"
greetings(name)
console.log("Bitti")
        
// İlk adımda fonksiyon tanımlanıyor greetings adlı fonksiyon kaydedilir.
// name adlı değişlen tanımlanmıştır.
// Bu adımda ilk adımda tanımlanan fonksiyon bu aşamada çalıştırılır.
               
// Çalıştırılan fonksiyon ekrana Merhaba Brendan çıktısını verir.
// Son aşama ile ekrana Bitti çıktı verilir ve işlem sonlanır.

JavaScript'in derlenmiş dil veya yorumlanmış dil olduğu konusunda tartışma mevcuttur. Ağırlıklı olarak derlenmiş dil fikri daha öne çıkmaktadır. Konu anlatıcı bu kısmı detaylı araştırma ve buna göre cevap verilmesi doğru olduğundan bu konu hakkında iddia da bulunmayıp kaynakça kısmında paylaşmıştır.

Event Loop (Olay Döngüsü)

Event Loop (Olay Döngüsü) kısaca Javascript'in asenkron işlemleri yapmaya yarayan sistemdir. JavaScript yapısı gereği tek işlem yapabilmektedir aynı anda iki işlemi yapamaz. Olay döngüsü ile ana işlemi engellemeden birden fazla görevin yapılmasını sağlar.

Call Stack

Call Stack (Çağrı yığını) JavaScript motorunun birden fazla işlevi çağıran koddaki yerini takip eder. 

function greet() {
    sayHi() // 2.adım
}
                            
function sayHi() {
    console.log("Merhaba Arkadaşım 🖐🏻") // 3.adım
}
                            
greet() // 1.adım
        
// greet fonksiyon çağrılır ve Call stack eklenir
// sayHi() fonksiyonu greet fonksiyonu çağrılır ve stack eklenir.
// console.log("Merhaba Arkadaşım 🖐🏻") çalışır
// sayHi() işlevi biter ve stack'ten çıkar
// greet() işlevi biter ve stack'ten çıkar
Web API (Tarayıcı API)

Web API (Tarayıcı API) tarayıcı tarafından sağlanan ortamın bir parçasıdır. JavaScript kullanarak tarayıcının özellikleri ile etkileşime girer. Web API özelliği JavaScript'in asenkron işlemlerin gerçekleştirmesini sağlar.

Task Queue (Görev Kuyruğu)

Task Queue (görev kuyruğu, görev listesi, veya mesaj kuyrukları) bekletilen işlemlerin sırasını tutar.

console.log("Merhaba")
            
setTimeout(() => {
    console.log("Ben setTimeout içindeyim")
}, 0) 

// Tarayıcı Web API gönderir asenkron işlemdir. Callback Queue (Geri Çağrı Kuyruğu) Asenkron kodun itilmesi ve yürütülmesini beklediği yer burasıdır.
            
console.log("Görüşürüz")
            
// Merhaba ve Görüşürüz hemen çalışır.
// 0 milisaniye ayarlanmış olsa bile Call Stack boşalana kadar bekler. setTimeout'un daha sonra yürütülmek üzere Task Queue kuyruğuna yerleştirmesi Web API olmasıdır.

Kodun Çıktısı bu şekilde olacaktır.

Merhaba
Görüşürüz
Ben setTimeout içindeyim
Microtask Queue (Mikro Görev Kuyruğu)

Mikro görev kuyruğu (Microtask Queue), mikro görevlerin listesini tutar. Microtask Queue yüksek önceliğe sahiptir. Promise ve MutationObserver gibi işlemler bu kuyruğa görev ekler. Bu görevler, olay döngüsünün (Event Loop) her yinelemesi tamamlandıktan hemen sonra yürütülür. 

console.log(1)

setTimeout(() => {
    console.log("2 (setTimeout)")
}, 0)
                                
Promise.resolve().then(() => {
    console.log("3 (Promise)")
})
                                
console.log(4)
Kod Çıktı
1
4
3 (Promise)
2 (setTimeout)

// console.log("1") ve console.log("4"), hemen çalışır çünkü bunlar call stack üzerindedir.
// Promise.then(...), Microtask Queue'ya gider ve call stack boşalınca hemen çalışır.
// setTimeout(...), Macrotask (Callback/Task Queue) kuyruğuna gider ve en sona bırakılır.
Asenkron Programlama

Eşzamansız Programlama ilgili programın çalışan programın bitmesini beklerken diğer kodları çalıştırmaya devam etmesidir.

console.log("1. İşlem başladı");

setTimeout(() => {
    console.log("2. Asenkron işlem (2 saniye sonra çalıştı)");
}, 2000);
                                                                
console.log("3. İşlem bitti");
                                                                
// setTimeout asenkron yapıdır. Gecikmeli çalışır.
Derleyici (Compiler)

Derleyici bir programlama dilini makine tarafından anlayacağı duruma çevirir. Çevirilen dil binary (ikili) biçimidir.

Yorumlayıcı (interpreter)

Bir programlama dilinde yazılmış kodu Derleyici kıyasla makine koduna dönüştürmeden yürütür. Kodları satır satır, sonraki satırı yürütmeden önce makine talimatlarına çevirir.

JIT (Just In Time)

JIT (Tam Zamanında Derleme) anlamına gelir. Kod çalışma zamanında ihtiyaç duyuldukça derlenir. JavaScript tarayıcıda çalışırken Yorumlayıcı (interpreter) tarafından satır satır çalışır. JavaScript Motorlarından Google'nin V8 Motoru;

  • Kod yorumlanır – Hızlı başlamak için hemen çalıştırılır.
  • Sık kullanılan kodlar tespit edilir – Bu kodlar "hot code" olarak adlandırılır.
  • JIT devreye girer – Bu sık kullanılan kısımlar makine koduna dönüştürülür ve çok daha hızlı çalıştırılır.

JIT'ın optimasyonu hakkında ufak kod örneği;

function multiply(a, b) {
    return a * b
}

console.time("İlk Çalışma")
for (let i = 0; i < 1_000_000; i++) {
    multiply(10, 20)
}
console.timeEnd("İlk Çalışma")


console.time("İkinci İşlem")
for (let i = 0; i < 1_000_000; i++) {
    multiply(10, 20)
}
console.timeEnd("İkinci İşlem")

// İlk aşamada Multiply fonksiyonu çağrılıp çalışmakta
// Tarayıcı Multiply fonksiyonu tekrarlandığını fark eder (hot code)
// JIT devreye girerek fonksiyonu optimize edip makine koduna çevirir bu sayede kod hızlı çalışır.
// İkinci kod bloğu daha hızlı çalışacaktır.
JavaScript Motoru

JavaScript motoru (JS motor) JavaScript kodunu yorumlar ve yürütür. Modern JavaScript motorları, JavaScript kodunu bilgisayar işlemcisi tarafından yürütülen makine koduna dönüştürmek için JIT kullanır. JavaScript motoru genellikle istemci tarafında kodu çalıştırmak için kullanılır. Node.js tarafında da kullanılabilir. Birden çok JavaScript motoru bulunmaktadır. Çoğunlukla ön plana çıkan JavaScript motorları şunlardır; Diğer motorlar hakkında detaylı bilgi için buradan inceleyebilirsiniz

  • V8 (Google)
  • JavaScriptCore (Apple)
  • SpiderMonkey (Mozilla)
  • Presto (Opera (2013 yılı itibari ile Blink Motorunu kullanmaktadır.))
  • Chakra (Internet Explorer - Microsoft Edge (Microsoft Edge Chromium tabanlı geçiş yapıldı kaynakça bölümünde haberi inceleyebilirsiniz))

Node.Js Neden Gerekli ?

JavaScript, başlangıçta tarayıcıda çalışmak üzere geliştirilmiştir. Ancak Node.js sayesinde JavaScript'i sunucu tarafında da kullanmak mümkün hale geldi. Node.js ile tarayıcıdan bağımsız çalışan JavaScript uygulamaları geliştirilebilir.

Node.js’in modül sistemi (ileride detaylı anlatılacak) ve ECMAScript sürüm desteği, geliştiricilere büyük esneklik sağlar. Modern web projelerinde NPM gibi bağımlılık yönetim sistemleri kritik önemdedir. Bu sayede JavaScript kodları hem istemcide (client-side) hem de sunucuda (server-side) çalıştırılabilir.

Ayrıca, React ve benzeri framework’lerle çalışırken; Webpack, Babel, ESLint gibi araçlar Node.js ile entegre çalışır. Özellikle büyük ölçekli projelerde kurulum, derleme (build), test ve güncelleme süreçlerinde büyük kolaylık sağlar.

Yorum Satırı

Kod yazarken bazı kod bloklarını yürütmemesi için, veya ilgili kod ile alakalı bilgi verirken kullanılması önemlidir. Yorum satırına alınan kod yürütülmez. Tek satırlı JavaScript'te tek satırlı yorum satırı // çok satırlı yorum satırı /* */ işareti ile yapılır. Aşağıda gösterilmiştir.

console.log("Merhaba Dünya 🖐🏻")

Tarayıcıda JavaScript Çalıştırma

Tarayıcıda JavaScript çalıştırmak için HTML sayfası içine .js uzantılı JavaScript dosyası eklenmelidir. HTML içinde JavaScript 3 farklı şekilde eklenebilir bunlar;

  • Harici JavaScript (External Script)
  • Dahili JavaScript (Internal Script)
  • Satır içi JavaScript (Inline Script)

JavaScript kodunu tarayıcıda görüntülemek için Ctrl + Shift + J ve/veya F12 tuş kombinasyonu üzerinden tarayıcı konsolu açılır. Websitesinin hızına etki etmemesi için HTML sayfalarında body etiketi kapanmadan önce yerleştirmeniz daha iyi olacaktır. Bir sayfada birden fazla harici JavaScript dosyasını ekleyebilirsiniz.

Bilgi

Kodların bulunduğu JavaScript dosyalarını sayfanın sonundan indirebilirsiniz.

Node.Js Üzerinde Çalıştırma

JavaScript'i Node.Js üzerinde çalıştırması mümkündür. Node.Js üzerinde çalıştırmak için Node.Js kurulu olmalıdır. Node.Js üzerinden çalıştırmak için komut satırından node ilgili dosyanın adını yazmanız yeterlidir. (Aşağıdaki örnekte dosya ismi örnek olarak verilmiştir. Kendi bilgisayarınızda çalıştırırken app yerine adlandırdığınız dosya adını yazmalısınız.)

node app.js
